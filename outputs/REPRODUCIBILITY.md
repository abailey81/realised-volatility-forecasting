# Reproducibility

Every result table and figure in `outputs/` is reproducible from the
raw minute-bar files in `data/raw/` and the public macro sources
listed below. This file is the exact step-by-step recipe. If a third
party runs these commands on a fresh checkout, they will obtain
numerically identical results up to bootstrap variance.

---

## 1. Environment

| Item | Version |
|---|---|
| OS tested | macOS 25 (Darwin 25.3.0), Linux Ubuntu 22+ |
| Python | 3.11 or 3.13 |
| Package manager | pip |
| Free disk space | ≥ 1 GB |
| RAM | ≥ 8 GB (16 GB recommended for the NN training) |
| Cores | ≥ 4 (16 recommended for parallelism) |
| Network | required for macro feature download only |

### 1.1 Install

```bash
git clone <repo>
cd rv_ml_replication-2
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

Pinned versions in `requirements.txt`:

* numpy ≥ 1.26
* pandas ≥ 2.0
* scikit-learn ≥ 1.4
* scipy ≥ 1.11
* statsmodels ≥ 0.14
* matplotlib ≥ 3.8
* yfinance ≥ 0.2.30
* pyyaml, joblib, tqdm (development)

### 1.2 Optional environment variables

```bash
# Required for downloading FRED-sourced macro features (free key, ~1 min):
export FRED_API_KEY=<your key>
# (If you skip this, the FRED-CSV fallback in src/data/macro_features.py
# will be used, which does not require an API key.)
```

### 1.3 Tests

```bash
python -m pytest tests/ -v
# Expect: 37 passed
```

## 3. End-to-end pipeline (single command)

```bash
python scripts/00_run_full_pipeline.py
```

This drives stages 1–9 in order. Total runtime on a 16-core M-series
laptop: ≈ 1 h 15 min. The longest single stage is `06_train_nn.py`
(≈ 40 min), all other stages run in 2–5 min each.

## 6. Configuration knobs (for faster dev iterations)

`config/config.yaml` defaults to the paper-faithful settings. For
faster iteration, clone it:

```bash
cp config/config.yaml config/config.dev.yaml
```

and edit the dev copy:

```yaml
models_nn:
  ensemble:
    num_random_seeds: 5         # was 100  → 20× faster NN training
mcs:
  num_bootstrap: 1000           # was 10000 → 10× faster MCS
```

Then run with `--config config/config.dev.yaml` (any script reads the
`load_config()` default if not overridden; set the env var
`RV_CONFIG=config/config.dev.yaml` to override).

## 7. Determinism

* `set_global_seed(42)` is called at the top of every script via
  `src/utils.py:set_global_seed`. It seeds Python's `random`, numpy,
  and sklearn's random state.
* NN seeds are `[0, 1, ..., 99]` deterministic per architecture.
* Bootstrap replications use `np.random.default_rng(42 + offset)` per
  bootstrap call.
* Numpy BLAS/LAPACK threading is forced to 1 per worker via
  environment variables so that joblib parallel pools do not introduce
  scheduler-induced non-determinism.
* sklearn's `MLPRegressor` with a fixed seed is deterministic on a
  fixed-architecture, fixed-data setting.

There is one source of non-determinism we cannot fully eliminate: the
order in which joblib workers report results. We control for this by
sorting outputs alphabetically by (ticker, model, horizon) before
writing tables.

## 8. Validation

After the full pipeline runs, the following sanity checks should pass:

* `outputs/tables/` should contain ≥ 200 CSVs.
* `outputs/figures/` should contain ≥ 25 PDFs.
* `outputs/results/predictions_har_MALL.pkl` exists and contains 9
  `StockRunResult` objects (3 stocks × 3 horizons).
* `pytest tests/` reports 37 passed.
* On AAPL h=1 M_HAR the HAR MSE should be in the range [1.0e-8, 1.4e-8].
* On AAPL h=1 M_ALL the NN^10 MSE should be < HAR MSE (≈ 0.9× ratio).

## 9. Where to look when something goes wrong

| Symptom | Likely cause | Fix |
|---|---|---|
| `ModuleNotFoundError: src` | Forgot `sys.path.insert(0, ...)` | All scripts include it; check Python version |
| `MarkovRegression` fails with `SVD did not converge` | EM initialisation pathology | `fit_two_state_hmm` standardises + uses quantile-based start params and 25 random restarts — already fixed |
| `MLPRegressor` warns about non-convergence | Seed-specific | Increase `epochs` in config, or accept the warning (ensembling handles it) |
| MCS very slow | Default 10000 bootstraps | Drop `num_bootstrap` to 1000 in dev |
| QLIKE returns ∞ | NN prediction went negative | Already clamped at max(1e-12, min(y_train)·1e-3); inspect the offending forecast |
| RV looks too high | Outlier filter not run | Check `outlier_filter.enabled: true` in config |
| FRED download hangs | HTTP/2 issue in `requests` | Already fallen back to `curl --http1.1` in `src/data/macro_features.py` |
| ADS series 404 | FRED retired ADSWBCIND | Already replaced with Philly Fed XLSX backfill |

## 10. Hardware notes

The full pipeline was developed and tested on:

* macOS 25 (Darwin 25.3.0), Apple Silicon M-series, 16 cores
* Ubuntu 22.04 LTS, x86_64 16-core

The most memory-intensive step is the NN ensemble training: peak RSS
≈ 4 GB across all workers. With 8 GB RAM, set `n_jobs ≤ 4`. With 4
GB RAM, run the NN stage one architecture at a time using the
`--architectures` CLI flag.

No GPU is needed; sklearn's MLPRegressor is CPU-only and the
network depth (max 4 layers, max 16 hidden units) does not benefit
materially from GPU acceleration.
